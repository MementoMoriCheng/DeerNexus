## PR-023 — Backfill Job（存量资源回填默认 Org）

> Track B 第四项。PR-021（4 表可空 org_id）/ PR-022（默认 Org + admin 关系）已交付。本 PR 把存量 `org_id IS NULL` 的资源行（threads_meta / runs / run_events / feedback）批量回填到默认 Org。**隐式水位、批次限速、可重入、dry-run、行数/孤儿/空 org 校验**。纯数据回填——非 schema revision、非 enforce（NOT NULL 留 PR-025A）、不开 multi-org。

### 探索结论（关键事实）

1. **mapping = blanket → 默认 Org**（ADR-0002 §8.1 step 4「按资源依赖顺序回填默认 Org」），非 per-user 解析。系统 resolver 本就是 config-driven 单 Org（`default_org_id="default"`）；多 Org 留 PR-025B。
2. **回填顺序**（ADR-0002 §8.2）：threads → runs → run_events → feedback（4 表中 spec 列的其他资源 Memory/Artifact/Skill/MCP/Scheduler 表尚不存在）。4 表互为**软引用无 FK**（runs.thread_id、run_events.run_id 等都无 ForeignKey）→ 每表独立 `UPDATE ... WHERE org_id IS NULL`，不需 join。
3. **隐式水位**（用户确认）：查询谓词 `WHERE org_id IS NULL` 本身就是水位——已回填行被排除；每批 commit 后崩溃/重跑自然从最后提交点恢复。进度 = 每批后日志报告剩余 NULL 行数。无需进度表（ADR §8.2 的「进度表」属生产规模加固，testing-strategy §15.3 单独跟踪）。
4. **不在应用启动自动执行**（pr-split-guide §14「依赖应用启动自动执行长迁移」禁止）→ 显式 CLI 作业，不挂 lifespan。
5. **reset_admin.py = 生命周期模板**（`asyncio.run` + deferred imports + `init_engine_from_config` → `get_session_factory()` → work → `close_engine()` in finally）；**migrate_user_isolation.py = dry-run/报告/幂等重跑模板**（argparse `--dry-run` + report list + 重复运行 no-op）。
6. **无 batch-update 基础设施**：现有 repository 全是单行 UPDATE by PK；无 `executemany`/`update().where(<set>)`。PR-023 自建批量 `update(Table).where(org_id.is_(None)).values(org_id=default)` + 分页。
7. **前置条件**：默认 Org 必须存在。`init_engine_from_config` 只 bootstrap schema，不跑 lifespan 的 `_ensure_default_org`。回填 CLI 先幂等调 `ensure_default_org`（防御，对齐 PR-022 helper）。
8. **校验目标**（testing-strategy §15.2）：迁移前后每表行数、orphan 数为 0、无空 org_id（系统白名单除外——4 资源表均不在白名单）、重复运行不重复创建。§15.3 规模验收（DDL lock/吞吐/事务日志）单独跟踪，非 PR-023 范围。
9. **§14 Migration-PR 标签集**（phase/compatible_app_versions/estimated_rows/...）是给 **schema revision** 的，PR-023 是数据回填作业非 revision——但 CLI 会在输出里报告 estimated_rows/batch_size/throttle/validation_queries 等量，便于运维对照 runbook。

### 设计决策

- **核心逻辑放 harness 层可测**：`deerflow/tenancy/backfill.py` 提供 async 函数 `backfill_resource_org(sf, *, org_id, batch_size, throttle_ms, dry_run) -> BackfillReport`，CLI 仅薄 shim。对齐 PR-022 把 helper 放 harness 的惯例，且可被 `init_engine`+直调驱动测试，不耦合 FastAPI。
- **单命令全 4 表**（用户确认），内部按依赖序 threads→runs→run_events→feedback。`backfill_default_org [--dry-run] [--batch-size N] [--throttle-ms M] [--org-id ID]`。
- **隐式水位 + 批次限速**：循环 `UPDATE <table> SET org_id=? WHERE org_id IS NULL LIMIT <batch_size>`（SQLite 支持 UPDATE...LIMIT；Postgres 用子查询 `WHERE id IN (SELECT id ... LIMIT)`），每批 `await asyncio.sleep(throttle_ms/1000)`，`result.rowcount` 报告本批影响数，0 则该表完成。每批独立 session+commit（崩溃恢复粒度 = 1 批）。
- **dry-run**：不 UPDATE，只 `SELECT count(*) WHERE org_id IS NULL` per table，报告候选行数 + 目标 org + 预计批次，退出。对齐 migrate_user_isolation 的 `--dry-run` report 形态。
- **BackfillReport 数据类**：per-table `{before_null_count, updated_rows, after_null_count, batches}`；汇总 `total_updated, validation`。dry-run 与真实运行同结构（dry-run 的 updated=0）。
- **校验（validate post-backfill，非 dry-run 时）**：
  - **行数守恒**：每表 backfill 前后 total 行数不变（只改 org_id，不增删行）。
  - **orphan=0 / 无空 org_id**：每表 `SELECT count(*) WHERE org_id IS NULL` == 0（4 表均非白名单）。
  - **FK 完整**：`SELECT count(*) WHERE org_id NOT IN (SELECT id FROM organizations)` == 0（RESTRICT FK 已强制，但显式校验作为 acceptance gate）。
  - **重复运行幂等**：第二次运行 updated_rows==0（WHERE NULL 无候选）——这是测试断言点。
- **前置 ensure_default_org**：CLI 开头幂等调 `ensure_default_org(sf, org_id=config.default_org_id, slug, name)`（PR-022 helper）。若 default Org 缺失且 ensure 失败→退出码 1 + 清晰错误。
- **默认 org_id 来源**：`get_gateway_config().default_org_id`（与 PR-022/PR-013 resolver 一致）；`--org-id` 覆盖（运维灵活）。
- **audit event**：回填开始/结束 emit `backfill_started`/`backfill_completed`（org_id=default, payload=report summary），复用 PR-022 的 `emit_tenant_event`（临时接口，不静默）。
- **CLI 放 `backend/scripts/backfill_default_org.py`**（对齐 `migrate_user_isolation.py` 同类一次性数据迁移位置）；核心逻辑 `deerflow/tenancy/backfill.py`。

### 改动清单

**1. 新增 `backend/packages/harness/deerflow/tenancy/backfill.py`**（核心逻辑，harness 层）
- `@dataclass BackfillTableResult`：`table, before_null_count, updated_rows, after_null_count, batches`。
- `@dataclass BackfillReport`：`org_id, dry_run, tables: list[BackfillTableResult], total_updated, validation: dict`。
- `RESOURCE_TABLES`（依赖序）：threads_meta → runs → run_events → feedback。每表配其 model + PK 列（用于 LIMIT 子查询）。
- `async def backfill_resource_org(sf, *, org_id, batch_size=500, throttle_ms=50, dry_run=False) -> BackfillReport`：
  - dry_run：每表 `SELECT count(*) WHERE org_id IS NULL`，填 before_null_count，updated=0。
  - 真实：循环 `UPDATE <table> SET org_id=:org WHERE org_id IS NULL` + LIMIT（SQLite 直接 LIMIT；通过 `inspector.dialect` 判断，Postgres 用 `WHERE <pk> IN (SELECT <pk> ... LIMIT :n)`）；每批 commit + sleep(throttle)；rowcount==0 则下一表。
  - 校验：backfill 后每表 total 行数（守恒）、after_null_count（==0）、FK 完整（org_id 不在 organizations 的行数==0）。
- 幂等：WHERE NULL 天然；重复运行 updated==0。
- 不 import app（harness-boundary）；org_id 由 caller 传。

**2. 新增 `backend/scripts/backfill_default_org.py`**（CLI shim）
- argparse：`--dry-run`、`--batch-size`（默认 500）、`--throttle-ms`（默认 50）、`--org-id`（默认 config.default_org_id）。
- 生命周期镜像 reset_admin.py：`asyncio.run(_run(args))`；deferred imports；`init_engine_from_config` → `get_session_factory()` → `ensure_default_org`（前置）→ `backfill_resource_org` → print report → `close_engine()` in finally；`sys.exit(0/1)`。
- 报告输出对齐 migrate_user_isolation：per-table INFO 日志 + 汇总（total_updated, validation pass/fail）；dry-run 标注「DRY RUN」。

**3. 新增 `backend/tests/test_backfill_default_org.py`**（幂等 + 校验测试）
镜像 test_default_org_bootstrap 约定（`@pytest.mark.anyio`、`sf` fixture 走 init_engine/close_engine、tmp_path、AsyncSession 直查）。builders：`_seed_org` + `_seed_resource_rows(table, *, with_org, count)`（插带/不带 org_id 的行）：
- **TestDryRun**：插 NULL org 行，dry-run → report.dry_run=True，updated==0，before_null_count 正确，DB 未变（仍 NULL）。
- **TestBackfillFillsNullRows**：插 NULL + 非 NULL 混合行，回填 → NULL 行 org_id==default；非 NULL 行不变（不覆盖）。
- **TestIdempotentRerun**：回填 → 再回填 → 第二次 updated==0（WHERE NULL 无候选）。
- **TestValidationRowCountInvariant**：回填前后每表 total 行数不变（只改 org_id）。
- **TestValidationNoNullOrgId**：回填后每表 `count(*) WHERE org_id IS NULL` == 0。
- **TestValidationFkIntegrity**：回填后 `org_id NOT IN organizations` 行数 == 0。
- **TestDependencyOrder**：验证 4 表按 threads→runs→run_events→feedback 顺序回填（report.tables 顺序）。
- **TestBatchThrottle**：小 batch_size（如 2）+ 多行 → 多批，每批独立（updated_rows 累计正确）。
- **TestEnsureDefaultOrgPrecondition**：默认 Org 不存在时 backfill → ensure 创建它（不报错；FK 满足）。

### 不做（避免越界）
- **不 enforce NOT NULL**（→ PR-025A）——回填后 org_id 仍可空（列定义未变），只是存量 NULL 被填满。
- **不开 multi-org**（→ PR-025B）——单默认 Org。
- **不做 repository org scope / 隔离矩阵**（→ PR-024A–E）。
- **不做进度表 / 生产规模验收**（testing-strategy §15.3）——隐式水位足够单 Org 迁移。
- **不回填 Memory/Artifact/Skill/MCP/Scheduler/Checkpoint**（表不存在 / 归 LangGraph）。
- **不在应用启动自动执行**（§14 禁止）——显式 CLI。
- **不动 doctor**（→ PR-064）。

### 诚实标注
本 PR 是一次性数据回填作业——把存量 NULL org_id 行填到默认 Org，让 PR-025A 的 enforce NOT NULL 有零 NULL 前提、PR-024 仓储 org scope 有完整归属。**无运行时 org 过滤消费者**（PR-024）。隐式水位（WHERE NULL）对单 Org 一次性迁移足够；持久化进度表属生产规模加固（§15.3）。audit event 复用 PR-022 临时接口（PR-041 outbox 替换）。回填不改 schema（org_id 列仍可空），只改数据。合并后文档 PR 标记 PR-023 已交付，PR-024/PR-025A 前置条件进一步就绪。
